Sharing scientific tools from script to desktop

Mon 11 July 2022
By Jonathan Rocher Prabhu Ramach
  • YouTube

Description

The scientific Python ecosystem has been flourishing for more than 2 decades. It is very powerful and many tools can be built leveraging it, but one remaining challenge is how to disseminate this ecosystem deeper into industries and companies to make digital transformation a reality. While jupyter notebooks have made it easy to share scripts amongst regular programmers, many non-programmer scientists need a simple graphical user-interface to adopt new tools, technologies and practices.

Web based front-ends are certainly a way to solve this problem, but the client-server framework typically requires a shift in mental models for programmers, and may require more resources than may be readily available for small teams of scientists in early stages of their innovation process (IT, cloud services, front-end developers, security experts, â€¦). Desktop applications can represent a powerful alternative for small user base situations, proof of concepts, or applications that involve local datasets.

In this tutorial, we will teach you to migrate your script/notebook to an interactive and scalable GUI application which can be shared with users who are not comfortable with programming or command line tools. You will learn to re-architect your analysis so it can grow seamlessly into a robust, interactive GUI and even to a large analysis platform if need be. We will show the power of combining multiple powerful components of the open source Enthought Tool Suite (ETS) like TraitsUI, Chaco, and Mayavi. It is a mature, flexible and scalable GUI framework, used across academic and companies alike and across many industries and fields. By the end of the tutorial, attendees will all have built a simple interactive browser for pictures (dubbed ""pycasa""), and learned about the main features, capabilities, and principles of ETS.

Target audience: Programmers/developers wanting to share tools/algorithms with colleagues/scientists needing to focus on science rather than code.
